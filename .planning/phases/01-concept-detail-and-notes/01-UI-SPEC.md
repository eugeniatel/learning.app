## Spacing Scale

Declared values (multiples of 4, using rem in Tailwind):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (1) | Icon-to-label gaps, inline chip padding |
| sm | 8px (2) | Compact element spacing, tight gaps |
| md | 16px (4) | Default element spacing, card padding |
| lg | 24px (6) | Section padding, between content blocks |
| xl | 32px (8) | Layout gaps, between major sections |
| 2xl | 48px (12) | Major section breaks |
| 3xl | 64px (16) | Page-level top/bottom padding |
| shell-x | 40px (10) | Shell horizontal padding, inherited from session 1 |

Exceptions: shell-x (40px) is an inherited shell token from session 1. All other spacing uses the 8-point scale above.

## Color

Primary visual anchor: concept title (22px weight-500) at the top of the content column.

Source: globals.css :root tokens (session 1).

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | --background | #FAF9F5 | Page background |
| Secondary (30%) | --sidebar | #F5F3ED | Sidebar, right-rail panel background |
| Card surface | --card | #FFFFFF | Concept list item hover state, note editor surface |
| Muted surface | --muted / --secondary | #F1EFE9 | Open question list items, read-status chip, section divider fills |

Accent (#D97757) reserved for:
- Save button fill when the note editor has unsaved changes
- Focus ring on the note textarea (via --ring token, automatic)
- "Add question" button fill state (active/hover only)
- Active nav item indicator in SideNav when on /concepts route

Accent is NOT used for: headings, borders, read-status badges, artifact chips, section labels, module group labels.

## Component Inventory

### Reused from session 1 (no changes)

- `Shell` - outer layout with sidebar; max-w-3xl px-10 py-12 on main content
- `SideNav` - left rail nav; extend active state for /concepts route using coral ring or bg-sidebar-accent
- `ArtifactChip` - reused on concept detail unchanged. Already shows icon + title + type label. Read-status is not togglable in phase 1 (ARTIFACT-01 is phase 3). Show read status as a visual-only badge.

### New components for phase 1

**`ConceptCard`** (used in /concepts index)
- Container: full-width row, bg-card rounded-lg border border-border, px-4 py-3
- Concept title: 15px weight-500 text-foreground
- One-liner: 13px weight-400 text-muted-foreground, truncated to one line
- Module tag: 12px badge bg-muted text-muted-foreground, rounded-sm, no border
- Hover: bg transitions to muted (0.5px border stays, no shadow ever)
- Link: entire row is a next/link to /concepts/[slug]

**`ConceptDetail`** (composite, /concepts/[slug])
- Two-column layout: left content column (flex-1) + right sidebar (w-64 shrink-0), gap-8
- Left column sections in order: concept header, linked artifacts, notes, open questions
- Right sidebar sections in order: review stub, related concepts

**`ConceptHeader`** (inside ConceptDetail)
- Concept title: 22px weight-500, text-foreground, mb-1
- One-liner: 15px weight-400, text-muted-foreground, mb-0

**`ArtifactList`** (inside ConceptDetail)
- Section label: "Linked artifacts" - 13px weight-500 uppercase tracking-wide text-muted-foreground, mb-3
- Layout: flex-wrap gap-2
- Each item: ArtifactChip (existing component) + read-status badge appended
- Read-status badge: "read" or "unread", bg-muted text-muted-foreground text-[11px] rounded-sm px-1.5 py-0.5 - visual only in phase 1

**`NoteEditor`** (inside ConceptDetail, satisfies NOTE-01, NOTE-02)
- Section label: "Notes" - same section label style as above
- Textarea: full-width, min-h-[160px], bg-card border border-border rounded-lg px-3 py-2.5, text-[15px] weight-400 line-height-1.7, font-sans, resize-y
- Placeholder: "Write notes about this concept..."
- Save button: "Save note" - variant default (bg-primary text-primary-foreground) when dirty, variant outline when clean/saved
- Save state: button label changes to "Saved" for 2 seconds after successful save, then reverts to "Save note"
- Error state: inline text below textarea, text-destructive text-[13px], "Could not save. Try again."
- Layout: textarea full-width, save button right-aligned below textarea, gap-2

**`OpenQuestionCapture`** (inside ConceptDetail, satisfies QUESTION-01)
- Section label: "Open questions" - same section label style
- Existing questions: list of question text items, each in bg-muted rounded-md px-3 py-2 text-[14px] text-foreground, gap-1.5 between items
- Capture row: input (flex-1, placeholder "Park a question about this concept...") + "Add question" button (variant outline, size sm) side-by-side, gap-2, mt-3
- On submit: question appended to list optimistically, input cleared, writes to progress.json via server action
- Empty state: no list shown; only the capture input is visible

**`ReviewStub`** (right sidebar, phase 1 scope only)
- Section label: "Review" - same section label style
- Stub content: text-muted-foreground text-[13px] "Review available in the next phase."
- Border: bg-muted rounded-lg px-3 py-3, no interactive elements
- This is a presentation placeholder; no buttons, no actions. Phase 2 replaces this with ReviewCard.

**`RelatedConceptsList`** (right sidebar)
- Section label: "Related concepts" - same section label style
- Each item: concept title as next/link, text-[13px] text-foreground hover:text-foreground/70, block py-0.5
- Empty state: "No related concepts" text-muted-foreground text-[13px]

**`ConceptIndex`** (composite, /concepts)
- Page heading: "Concepts" - 22px weight-500
- Search input: full-width, placeholder "Filter by title...", mb-6, uses shadcn Input component
- Module group: module title as group label (13px weight-500 text-muted-foreground uppercase tracking-wide), followed by ConceptCard list, gap-1 between cards, gap-6 between module groups
- No results state: "No concepts match that title." text-muted-foreground text-[15px] mt-8

## Interaction Contracts

### Search on /concepts

- Input is client component with useState; filters ConceptCard list in-memory
- Filtering is case-insensitive title match
- No debounce needed (76 concepts, client-side array filter)
- URL is not updated on filter (no query params in phase 1)

### Note save (NOTE-01, NOTE-02)

- Textarea is a client component; tracks dirty state (initial value vs current value)
- Save is triggered by "Save note" button click only (no auto-save, no keyboard shortcut in phase 1)
- On save: calls server action in src/lib/notes.ts, writes notes/<slug>.md with frontmatter (conceptId, updatedAt) + body
- On load: textarea pre-populated from notes/<slug>.md if file exists; empty string if not
- Transitions: button opacity 180ms ease-out

### Question capture (QUESTION-01)

- Input clears on submit
- Question appended to progress.json.openQuestions via server action
- Each question gets id (uuid or timestamp), conceptId, text, status: "open", createdAt
- No confirmation dialog; action is low-stakes and reversible (Phase 3 adds park/answer actions)
- Pressing Enter in the input submits (same as clicking "Add question")

### Artifact read-status (visual only in phase 1)

- ArtifactChip shows read-status badge as text ("read" / "unread") - no toggle interaction
- Toggle is wired in Phase 3 (ARTIFACT-01)
- Badge renders from artifacts.json readStatus field
